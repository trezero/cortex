Section 5: Pattern Discovery Engine

Proactive workflow suggestion based on cross-repo activity analysis

Data Capture Pipeline

Three Input Streams → Unified Activity Events
Stream 1: Git Activity
Source: Post-commit hooks + periodic git log polling

Captured data:
• Commit message + diff stats
• Files touched (paths + languages)
• Branch name patterns
• Time of day / frequency
• Author (for multi-user setups)

Frequency: Real-time via hook, or every 15min poll
Stream 2: Agent Conversations
Source: remote_agent_messages table + Cortex chat sessions

Captured data:
• User request text (intent)
• Which repo / project scoped to
• Tools the agent used
• Workflow invocations triggered
• Outcome (success / failure / abandoned)

Frequency: Captured on conversation end
Stream 3: Workflow Runs
Source: workflow_runs + workflow_nodes tables

Captured data:
• Which workflow was run
• Node execution order + durations
• Which nodes failed / were skipped
• HITL approval patterns (always approved? always rejected?)
• Which repo + project context

Frequency: On workflow completion

Analysis Pipeline

Nightly Batch + On-Demand Analysis
STEP 1: INTENT EXTRACTION (per event)
For each new activity_event without an embedding:
  • Send event metadata to Anthropic API (haiku — cheap, fast)
  • Prompt: "Classify the developer intent in one sentence: {event_data}"
  • Store intent_summary + generate embedding via OpenAI/voyage
  • Example: commit "Add rate limiting middleware" → intent: "API security hardening"
  • Example: Telegram "run tests before merging" → intent: "Pre-merge validation"

STEP 2: CLUSTERING (nightly batch)
Query: activity_events from last 30 days, grouped by intent_embedding similarity
  • Use pgvector cosine similarity: 1 - (embedding <=> target)
  • Cluster threshold: similarity > 0.85
  • Minimum cluster size: 3 events across 2+ repos
  • Example cluster: "Pre-merge validation" appears in repos A, B, C
  •   - Repo A: user always runs "npm test && npm run lint" before PR
  •   - Repo B: user asks agent "run the test suite" before every merge
  •   - Repo C: workflow "pre-merge-check" exists, used 12 times

STEP 3: PATTERN SCORING
For each cluster, compute:
  • frequency_score = occurrences / days_in_window (how often?)
  • cross_repo_score = unique_repos / total_repos (how widespread?)
  • automation_potential = % of events that were manual (not workflow-triggered)
  • final_score = frequency × cross_repo × automation_potential
  • Threshold: final_score > 0.4 → candidate for suggestion

STEP 4: WORKFLOW GENERATION
For high-scoring patterns:
  • Send cluster data to Anthropic API (sonnet):
    "Given these repeated developer actions across repos, generate a
     reusable Cortex workflow YAML definition that automates this pattern."
  • Validate generated YAML against schema
  • Store in discovered_patterns table with status: 'pending_review'

Suggestion Surfacing

How Discovered Patterns Reach the User
Cortex UI — Suggestions Panel
New section on the Workflows page:

💡 Suggested Workflow
Pre-merge validation
Detected across 3 repos: you run tests + lint before every PR. This workflow automates that pattern.
Frequency: 4.2x/week · Repos: myapp, api-service, frontend
Accept Customize Dismiss
Accept → saves to workflow_definitions with origin: pattern_discovery
Customize → opens in workflow editor with pre-filled YAML
Dismiss → marks pattern 'dismissed', won't suggest again
Cortex Chat / MCP Integration
When user asks: "What should I automate?"
or: "Any workflow suggestions?"

Chat agent queries discovered_patterns table
and presents findings conversationally:

"I've noticed a pattern across your repos: you consistently run security audits before creating PRs in myapp, api-service, and frontend. This happened 14 times in the last month, always manually. Want me to create a workflow that automates this?"

MCP tool: cortex:suggest_workflows
Returns discovered patterns as structured data
IDE agents can present suggestions inline

Database Schema — Pattern Discovery

discovered_patterns table
Column Type Notes
id uuid PK
pattern_name text AI-generated descriptive name
description text Human-readable explanation of the pattern
cluster_embedding vector(1536) Centroid of the activity cluster
source_event_ids uuid[] FK → activity_events that formed this cluster
repos_involved text[] Repo URLs where pattern was observed
frequency_score float How often (occurrences/day)
cross_repo_score float How widespread (repos with pattern / total repos)
final_score float Composite ranking score
suggested_yaml text AI-generated workflow YAML
status text pending_review | accepted | dismissed | expired
accepted_workflow_id uuid | null FK → workflow_definitions (if accepted)
discovered_at timestamptz
